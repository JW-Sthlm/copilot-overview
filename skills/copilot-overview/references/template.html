The operating stack

Five layers, bottom-up. M365 Copilot is the foundation. Each layer builds on the one below.

👥 Squad — multi-agent orchestration optional · per-project
Persistent team of specialized AI agents in your repo.
Install: npm install -g @anthropic/squad then squad init
Team roles: Lead, Frontend, Backend, Tester, Scribe — each with a charter.md
State: All config in .squad/ — plain text, version-controllable
🤖 Agency — built-in agentic runtime built-in · no install
CLI spawns sub-agents in parallel. One orchestrator delegating to specialists.
Sub-agents: explore, task, general-purpose, rubber-duck, code-review
Fleet mode: /fleet — parallel execution for large tasks
Delegation: /delegate — hand off to GitHub Coding Agent
⚡ Copilot CLI — terminal-native interface ← this dashboard
Conversational AI agent in your terminal. Reads, writes, and executes code.
Extensibility: instructions, agents, skills, MCP servers, extensions, plugins
Modes: Normal, Plan, Autopilot
Install: winget install GitHub.Copilot · brew install copilot-cli · npm install -g @github/copilot
🐙 GitHub Copilot — the developer AI platform
GitHub's AI platform. Four surfaces:
Chat — VS Code, JetBrains, github.com sidebar
Coding Agent — assign Issues, get PRs back
CLI — terminal-native, agentic
Extensions — VS Code and github.com
Tiers: Pro, Pro+, Business, Enterprise. Quota pooled across surfaces.
🏢 Microsoft 365 Copilot — enterprise AI foundation click to expand ↓
Microsoft's AI layer across the productivity suite.
Surfaces: Teams, Outlook, Word, Excel, PowerPoint, SharePoint, Chat
Graph grounding: answers grounded in your emails, meetings, files, org
Bridge to CLI: WorkIQ plugin and M365 MCP tools bring M365 data into CLI sessions
Separate license from GitHub Copilot. Both active simultaneously via WorkIQ bridge.
Global — always loaded
Plugin — installed package
System — built-in
Project — per-repo

How the layers connect

Instructions = HOW to behave
Personality, rules, guardrails. Plain markdown. Learn more ↓
File format: .md (plain markdown)
Global: ~/.copilot/copilot-instructions.md
Project: .github/copilot-instructions.md, AGENTS.md, CLAUDE.md
How it works: Loaded silently into every prompt. The model reads these before your message.
Agents = PERSONA
Specialized roles invoked by name. No code. Learn more ↓
File format: .agent.md (markdown with YAML frontmatter)
Global: ~/.copilot/agents/*.agent.md
Project: .copilot/agents/*.agent.md
How it works: Invoked by name (e.g. @partner-briefing). The model becomes that persona.
Skills = KNOWLEDGE
Auto-triggered bundles with docs, templates, scripts. Learn more ↓
File format: SKILL.md + references/ + scripts/
Global: ~/.copilot/skills/*/SKILL.md
Project: .copilot/skills/*/SKILL.md
How it works: Auto-triggered by description match. Loads docs and scripts into context.
MCP Servers = CAPABILITIES
External APIs exposed as callable tools. Learn more ↓
Config: JSON in mcp-config.json or .mcp.json
Global: ~/.copilot/mcp-config.json
Project: .copilot/mcp-config.json or .mcp.json
How it works: Each server exposes tools over Model Context Protocol. Runs as separate processes.
Extensions = CODE
JavaScript hooks that intercept and transform prompts. Learn more ↓
File format: extension.mjs (ES module)
Global: ~/.copilot/extensions/*/extension.mjs
Project: .github/extensions/*/extension.mjs
How it works: Runs inside CLI process. Can modify prompts, register tools, process responses.
Plugins = BUNDLES
Installable packages from GitHub repos. Learn more ↓
Install: /plugin install owner/repo
Location: ~/.copilot/installed-plugins/
How it works: A plugin bundles agents, skills, and tools. All contents available in every session.

Scoping — what loads where

🌍 Global

Loads in every Copilot CLI session, regardless of cwd.

~/.copilot/

  • Instructions & voice profile
  • Custom agents
  • Global skills
  • All installed plugins
  • All MCP servers & extensions

📁 Project

Loads only when Copilot starts in that repo folder.

.copilot/ .github/

  • Project instructions & conventions
  • Project agents & skills
  • Per-repo MCP servers
  • Project extensions
  • Squad config (if present)
